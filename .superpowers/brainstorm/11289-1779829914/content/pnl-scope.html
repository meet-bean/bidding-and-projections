P&L scope: per-project or portfolio?

Should the P&L view show one project at a time, or roll up across all projects?

A

Per-project only

Select a project from a dropdown or list, see its P&L dashboard. One project at a time.

  • Simpler to build — just filter financials by project
  • Matches how the projection detail page works today
  • Enough for a demo with one project (Suncoast 3A)
B

Portfolio rollup + per-project drill-down

Default view: aggregated P&L across all projects. Click a project to see its individual P&L.

  • Shows the "big picture" — total company margin at a glance
  • More complex: need to sum financials across projects, handle different date ranges
  • Impressive for Mickey's boss (the budget approver)
C

Per-project now, portfolio later

Build per-project P&L first. Design the data model so portfolio rollup can be added without refactoring.

  • Ship something fast, iterate
  • Only one seed project (Suncoast 3A) so rollup would be underwhelming today
  • Avoids premature complexity while keeping the door open
