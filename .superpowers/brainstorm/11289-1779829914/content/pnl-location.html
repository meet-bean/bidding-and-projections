Where should the P&L view live?

This determines navigation structure and how users discover the feature

A

Dedicated /p-and-l route Recommended

Own top-level sidebar item, between Projections and Invoices. Project list that drills into per-project P&L detail.

  • First-class visibility — Superior is paying for this
  • Clean separation from cost forecasting (projections) and billing (invoices)
  • Room to grow: multi-project rollups, portfolio views later
B

Tab on projection detail page

Add a "P&L" tab alongside existing projection table/comments/alerts on /projections/:projectId.

  • Data is already on the page — just adds a financial lens
  • No new nav item needed
  • Downside: projection detail already has a lot going on
C

Section under Reports

Add P&L as a sub-tab within /reports, alongside equipment utilization and crew availability.

  • Groups all analytics together
  • Reports page currently only has operational metrics
  • Downside: buries a premium feature under a generic label
